## 🚀 使用方法

### 步骤 1：启动后端服务

```bash
python -m uvicorn backend.app:app --host 0.0.0.0 --port 8000 --reload
```

### 步骤 2：访问前端

打开浏览器访问：**http://localhost:8000**

### 步骤 3：进入自动标注页面

点击左侧菜单 **"自动标注"**

### 步骤 4：选择标注数量

在下拉框中选择：
- **10 张** - 快速测试，约 23 分钟
- **50 张（推荐）** - 日常标注，约 2 小时
- **100 张** - 批量标注，约 4 小时
- **200 张** - 大批量标注，约 8 小时
- **全部未标注图片** - 标注所有未标注图片

### 步骤 5：开始标注

点击 **"标注选中数量的图片"** 按钮

### 步骤 6：查看进度

后端日志会显示：
```
[INFO] [批量标注] 开始处理 50 张图片
[INFO] [标注进度] 1/50 - img_xxx
[INFO] [标注开始] 图片: ...
[INFO] [标注完成] 耗时: 5.23秒, 有效对象数: 3
...
[INFO] [批量标注完成] 成功: 50, 失败: 0
```

## ⚠️ 注意事项

### 1. 标注时间估算

| 数量 | 预计时间 | 说明 |
|------|----------|------|
| 10 张 | 约 23 分钟 | 快速测试 |
| 50 张 | 约 2 小时 | 推荐批次 |
| 100 张 | 约 4 小时 | 批量标注 |
| 200 张 | 约 8 小时 | 大批量 |
| 全部 | 根据未标注数量 | 自动计算 |

### 2. 显存占用

- Ollama 使用约 10GB 显存
- 剩余约 2GB 显存
- 标注过程中显存持续占用

### 3. 去重逻辑

- ✅ 已标注的图片不会被重复标注
- ✅ 每次标注前会检查已标注列表
- ✅ 标注完成后自动更新未标注数量

### 4. 错误处理

- 单张图片标注失败不影响其他图片
- 失败的图片会记录详细错误原因
- 可以后续重新标注失败的图片
