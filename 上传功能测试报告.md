## 🔍 400 Bad Request 可能原因

如果您在前端遇到 400 错误，可能是：

### 1. 文件未正确选择
**解决方案**: 
- 点击上传区域或"选择文件"按钮
- 确保选择了图片文件
- 检查文件列表是否显示已选择的文件

### 2. 文件格式不支持
**支持的格式**: 
- ✅ .jpg
- ✅ .jpeg  
- ✅ .png
- ✅ .bmp
- ✅ .webp

**不支持的格式**:
- ❌ .gif
- ❌ .svg
- ❌ .ico
- ❌ 其他非图片格式

### 3. 浏览器缓存问题
**解决方案**:
```
按 Ctrl+Shift+Delete 清除缓存
或按 Ctrl+F5 强制刷新页面
```

### 4. 文件为空
确保选择的文件不是空文件（0 字节）

## 📋 正确的使用步骤

### 步骤 1: 启动后端
```bash
python -m uvicorn backend.app:app --host 0.0.0.0 --port 8000 --reload
```

### 步骤 2: 访问前端
打开浏览器：**http://localhost:8000**

### 步骤 3: 上传图片
1. 点击左侧菜单 **"图片上传"**
2. 点击上传区域或拖拽图片到上传区域
3. 确认文件列表显示已选择的文件
4. 点击 **"开始上传"** 按钮
5. 等待上传完成

### 步骤 4: 验证上传
- 查看页面下方的图片列表
- 新上传的图片应该显示在列表中
